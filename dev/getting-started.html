Getting Started

This guide covers the full setup, local development workflow, and GitHub Pages deployment for Mini Wiki.

Prerequisites

Mini Wiki has no build dependencies. You need only a text editor and a browser. For local development with full functionality (search, Desmos), a local HTTP server is recommended:

# Python 3 (built-in)
python -m http.server 8080

# Node.js (via npx, no install required)
npx serve .

Then open http://localhost:8080 in your browser.

Adding a New Article

Follow these steps for every new page:

Step 1 — Copy the template

Copy pages/article-template.html to the appropriate subfolder:

cp pages/article-template.html pages/math/my-topic.html
Step 2 — Fill in the content

Edit the new file. Update:

  • The <title> tag
  • The <h1> heading
  • The breadcrumb trail
  • The window.WIKI_ROOT = '../'; window.WIKI_PAGE object (title, category, tags)
  • The window.WIKI_SIDEBAR object
  • All article content

Adjust the relative paths for CSS and JS: ../../assets/css/wiki.css for pages in subdirectories.

Step 3 — Register in the search index

Open index.html and add an entry to window.WIKI_SEARCH_PAGES:

{ title: 'My Topic',
  url:   'pages/math/my-topic.html',
  desc:  'A short description for the search results.',
  tags:  ['math', 'keyword'] },
Step 4 — Link it from the sidebar

Add the page to window.WIKI_SIDEBAR in related articles so users can discover it through navigation.

Deploying to GitHub Pages

  1. Create a public GitHub repository.
  2. Push the contents of the wiki/ folder to the repository root (or a subdirectory).
  3. Go to Settings → Pages.
  4. Set Source to Deploy from a branch, select main and / (root).
  5. Click Save. The site will be live at https://yourusername.github.io/repository-name/ within a few minutes.
Subdirectory Deploy

If deploying to a subdirectory (e.g. /wiki/), update WIKI_CONFIG.rootUrl in wiki.js from './' to '/wiki/' so all header navigation links resolve correctly.
